## 📡 第二阶段：多端客户端配置 (V2rayN / 小火箭)

这套配置核心在于**“偷梁换柱”**。复制美国原始节点信息，然后修改地址。

### 🖥️ Windows 端：v2rayN 配置
1. 复制美国 VPS 上的原始 Reality 或 Hysteria 2 节点链接，导入客户端。
2. 双击打开该节点配置，进行核心重构：
   * **地址 (Address)：** 修改为 👉 **台湾 VPS 的公网 IP**。
   * **端口 (Port)：** 填写你在台湾上设置的监听端口（如 `18607`）。
   * **伪装域名 (SNI) / UUID / 公钥pbk：** **全部填写美国端原始配置，绝对不能动**！
3. 保存并连接。你的流量会秒连台湾，瞬间跳跃到美国。

### 📱 iOS 端：Shadowrocket (小火箭) 配置
1. 先复制美国节点的原始链接。
2. 点击节点右侧的 ℹ️ 详情图标：
   * **地址：** 删掉美国 IP，填入 👉 **台湾 VPS 的公网 IP**。
   * **端口：** 填入 `18607`。
   * **UUID / 密钥 pbk / TLS 设置：** 保持原样不动。
3. 点击完成，开启小火箭。你会在流媒体秒开中感受到台湾中转的魅力。

## 💰 第三阶段：专业跨境深度伪装 (ADS 指纹浏览器 + 静态ISP落地)

如果你是帮别人做 **TikTok 养号、Amazon 测评**，需要用到买来的**美国静态住宅 ISP（ISP Proxy）**。

**⚠️ 大佬的技术训话：**
如果你跑跨境电商，**绝对不能通过代理软件开启全局代理**！那纯粹是在侮辱各大平台的风控系统。正确做法是抛弃客户端加速，直接在 ADS 指纹浏览器内部完成**“物理对接”**！

假设你买了一个 SOCKS5 住宅代理账号（如 `198.51.100.1:9999`, 账号: `ispuser`, 密码: `isp_pass`）。

**1. 台湾中继配置补丁：**
在**台湾 VPS** 上，监听一个本地端口（例如 `8888`），映射到那个买来的静态 ISP 代理地址：
```bash
# 台湾 VPS 上启动转发到 ISP 代理
nohup ./realm -l 0.0.0.0:8888 -r 198.51.100.1:9999 > /dev/null 2>&1 &
```

**2. AdsPower (ADS) 深度伪装配置：**
打开 ADS 代理设置：
* **代理类型：** SOCKS5。
* **代理主机：** **台湾 VPS 公网 IP**（此时台湾 VPS 就是一个加速网关）。
* **代理端口：** 填入刚才台湾上设置的 `8888`。
* **账号/密码：** 填写你买的那个美国静态住宅代理的凭证 (`ispuser` / `isp_pass`)。
* **检查代理：** 点击测试。如果是绿色的“连接成功”，物理链路已彻底打通！

> 💡 **大佬的结案陈词：** 这套配置下，你的指令通过台湾跳板直达美国家宽，WebRTC 和 DNS 均由美国落地端解析，环境纯度 100%。
