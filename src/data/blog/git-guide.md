折腾了一大圈，终于把我的 Astro 博客搭建好了，并且成功在本地建立了完美的克隆副本。

很多新手（包括我自己）刚开始用 GitHub 搭建博客时，最头大的就是怎么把电脑上写好的文章同步上去。其实，只要你配置好了环境，这一套流程非常简单，只需要记住这三个“魔法咒语”。

## 📦 Git 同步三部曲

想象一下我们在**淘宝购物**的过程，Git 的操作其实一模一样。

### 第一步：打包（git add）

当你写完文章或修改了代码后，这些文件还只是散落在你的“房间”里。我们需要把它们捡起来，放进“快递盒”里。

打开 VS Code 的终端，输入：

```bash
git add .
```

* **注意**：`add` 后面有个空格，然后是一个点 `.`。
* **含义**：这个点代表“全部”。意思就是：**“把我刚才修改的所有东西，都放进快递盒。”**

---

### 第二步：贴单（git commit）

东西放进盒子里了，但快递员不知道这盒子里装的是啥，也不知道要发给谁。我们需要给快递盒封口，并贴上一张备注单。

输入：

```bash
git commit -m "写了一篇新文章"
```

* **注意**：`-m` 后面跟着的是双引号。
* **含义**：`commit` 是“提交/确认”的意思。引号里的字是你对这次修改的备注。
* **建议**：备注最好写清楚，比如“修改了Logo”、“新增了友链”，方便以后查账。

---

### 第三步：发货（git push）

盒子打包好了，单子也贴了，最后一步就是把快递交给快递员（GitHub），让他发往云端服务器。

输入：

```bash
git push
```

* **含义**：`push` 就是“推”。把本地的代码用力推到 GitHub 仓库里。
* **效果**：当你按下回车，看到进度条跑完的那一刻，你的博客源码就更新了。几分钟后，Cloudflare 就会自动把你的新文章部署到全世界。

## ⚡️ 进阶小技巧：网络加速

如果你在国内，`git push` 经常会卡住或报错（Connection reset），那是因为连接 GitHub 的线路不稳定。

如果你有“魔法”工具（代理软件），可以在终端里输入这两行命令，给 Git 开个加速挂（注意端口号要改成你软件的端口，比如 7890 或 10808）：

```bash
git config --global http.proxy [http://127.0.0.1:10808](http://127.0.0.1:10808)
git config --global https.proxy [http://127.0.0.1:10808](http://127.0.0.1:10808)
```

设置一次，永久有效。从此以后，`git push` 就是秒传！

## 总结

以后写博客的流程就是：
1. **写文章**（在 VS Code 里写）
2. **打包**：`git add .`
3. **贴单**：`git commit -m "update"`
4. **发货**：`git push`

搞定收工！🚀
